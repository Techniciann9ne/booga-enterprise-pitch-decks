# Booga Agents content governance (Phase-1 public rules)

These are content rules, not code — a future agent editing the deck cannot derive them from the source and must not violate them. Authoritative source: the client build-instruction file in `attached_assets/` (§2, §8 QA checklist).

## Naming (non-negotiable)
- Never the bare word **"Booga"** alone; never **"Buga"**.
- Product = **Booga Agents**. Company / byline / footer = **Booga Enterprise**. "Booga One" only in the one scale-up line.
- The autonomous things that run are lowercase **agents** (e.g. "Agents draft, professionals decide"), never "Booga agents" as a product name.

## Forbidden content
- No **"SOC"** anywhere. Compliance shown as *reporting support for* SOX / GDPR / HIPAA / ISO 27001 / PCI DSS — never "certified".
- No financial vertical, no funding / ARR / runway / team-size / solo-founder, no predecessor entity.
- No named competitors — comparison uses generic alternatives only (Generic AI copilots / Build it yourself / Point tools).
- No invented customers, metrics, quotes, logos, regions, or capabilities. Open items render as visibly-styled editable `[bracket]` placeholder tokens.
- **Pricing never appears on the live deck or teaser** — confidential leave-behind only.

## Citable facts (use only where called for)
- 600+ documented REST endpoints; **11 plugins** (the word "plugins", never "modules"); 7-year default audit retention; tenant-isolated infra (own DB, own vector index, own region); multilingual document extraction.
- Byline: **Dr. Mario Baburić, Founder & CEO, Booga Enterprise**.

## Brand system
- Light-first (overrides any dark template). Light `#F6F6F6` bg, Stone `#BEBEBE`, Blackout `#121212` text, Light Green `#74E4B0`, Vibrant Pop `#00AA5B` (the accent, one per slide, sparing), Deep Base `#145738`. Inter only. One sparing dark slide max. Spine motif: Connect → Understand → Act wrapped in Trust. No photos of people/robots/brains/handshakes.
